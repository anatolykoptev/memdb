# MemDB v0.22.0 — First Public Release

**2026-04-26**

This is MemDB's first public release. Earlier v1.x and v2.x tags were internal
pre-public iterations. See [docs/versioning.md](../versioning.md) for the
re-versioning rationale.

---

## Quick Start

```bash
# Clone and start
git clone https://github.com/anatolykoptev/memdb.git
cd memdb
cp docker/.env.example docker/.env  # fill OPENAI_API_KEY or LLM_API_BASE
docker compose -f docker/docker-compose.yml up -d

# Verify health
curl http://localhost:8001/health
# {"status":"ok","service":"memdb-mcp","version":"0.22.0"}

# Add a memory
curl -X POST http://localhost:8080/product/add \
  -H "Content-Type: application/json" \
  -d '{"user_id":"demo","cube_id":"demo","messages":[
        {"role":"user","content":"My name is Alice and I work at Acme Corp"}
      ]}'

# Search memories
curl -X POST http://localhost:8080/product/search \
  -H "Content-Type: application/json" \
  -d '{"user_id":"demo","cube_id":"demo","query":"where does the user work?"}'
```

## Architecture

MemDB v0.22.0 runs as a **pure-Go stack** — no Python in the hot path.

```
┌─────────────────────────────────────────────────────────────┐
│  Client (REST / MCP)                                        │
└─────────────────────┬───────────────────────────────────────┘
                      │
         ┌────────────▼────────────┐
         │     memdb-go (:8080)    │  Go/Axum — add, search, chat
         │  (+ memdb-mcp :8001)   │  MCP server — Claude Code etc.
         └────────────┬────────────┘
                      │
     ┌────────────────┼─────────────────┐
     │                │                 │
┌────▼─────┐   ┌──────▼──────┐  ┌──────▼──────┐
│ Postgres │   │    Redis     │  │  embed-srv  │
│ 17 + AGE │   │  8.6 VSET   │  │  ONNX e5-L  │
│ pgvector │   │  hot cache  │  │  1024-dim   │
└──────────┘   └─────────────┘  └─────────────┘
```

**Containers (6 total):**
`postgres` · `redis` · `qdrant` · `embed-server` · `memdb-go` · `memdb-mcp`

Python `memdb-api` was removed 2026-04-26 (M9 Stream 8, PR #93). Zero regressions
after 2-day dry-run with Python container idle.

## Why Now — What Shipped in This Release

### Pure Go runtime

6 containers (was 7). Python `memdb-api` is gone. All memory pipelines
(add/search/chat/scheduler) run natively in Go.

See [ROADMAP-GO-MIGRATION.md](../../ROADMAP-GO-MIGRATION.md) for the full
phase-by-phase migration history.

### Memobase-comparable measurement (M9)

Four features from the Memobase paper (see `docs/competitive/2026-04-26-memobase-deep-dive.md`)
have been ported and measured on LoCoMo:

| Feature | PR | Effect |
|---------|----|--------|
| Dual-speaker retrieval | #92 | Closes 32% attribution-suppression class (cat-5) |
| LLM Judge metric | #91 | Apples-to-apples scoring vs Memobase/Mem0/Zep public numbers |
| Cat-5 exclusion + dual-track reporting | #88 | Honest comparable aggregate F1 |
| `[mention DATE]` time-anchoring in extract prompt | #90 | cat-3 temporal lift |

**Aggregate LoCoMo F1 trajectory**: 0.053 (baseline) → 0.238 (M7, +349%) → measured in M9 with LLM Judge metric.

### Phase D retrieval — fully shipped (D1-D11)

All ten Phase D features from [docs/backlog/search.md](../backlog/search.md) are
in production:

- D1: importance-based rerank + decay
- D2: multi-hop graph expansion (AGE Cypher traversal)
- D3: relation detector — CAUSES/SUPPORTS/CONTRADICTS edges
- D4: session boundary detection
- D5: temporal weight (valid_at recency bias)
- D6: cross-encoder rerank (CE)
- D7: query expansion (LLM) with recall hint
- D8: scope-aware fanout (user/session/global)
- D9: SearXNG internet retrieval
- D10: fine-mode LLM filter
- D11: CoT query decomposition for multi-hop + temporal questions

## Versioning Reset

| | Version | Date |
|-|---------|------|
| Previous internal | v2.2.0 | 2026-04-26 morning |
| **Public** | **v0.22.0** | **2026-04-26 evening** |

API and wire format are **unchanged**. The reset reflects that pre-public
iterations are not committed-to stable contracts.

**Update your image tag:**
```yaml
# Before
image: anatolykoptev/memdb:v2.2.0

# After
image: anatolykoptev/memdb:v0.22.0
```

Schema migrations run automatically on `memdb-go` startup via `postgres_migrations.go`.

Full rationale: [docs/versioning.md](../versioning.md).

---

## Migration from v2.x

1. Update image tag in your compose file (see above)
2. No code changes required for REST API or MCP clients
3. Schema auto-migrates on first startup — no manual `psql` needed

---

## Breaking Changes

None at the on-the-wire level.

The version reset itself is the only "breaking" change — `^v2` semver ranges no
longer satisfy the latest release. Pin to `v0.22.0` or use `^v0.22`.
